Docker 概述

Docker为什么会出现？

环境：开发环境、线上环境

角色：开发同学、运维同学

版本：升级、发版

配置：集群部署、单点部署、重复的配置

交付：简化、方便

Docker思想来自于集装箱，隔离是它的核心思想。

隔离机制将服务器利用到极致。

Docker的历史

2010 dotCloud paas云计算技术 LXC 有关的容器技术

统一简化命名 Docker

最初诞生没有引起行业注意，想做开源

2013 Docker 开源 逐渐开始引起行业注意 每个月更新一个版本

2014-4-9 Docker1.0发布

Docker十分轻巧，与虚拟机技术相比更优。

虚拟机：windows Vmware 虚拟多台电脑 笨重 也是虚拟化技术

容器：docker容器技术也是虚拟化技术

vm linux centos 原生镜像

docker 隔离 镜像 十分小巧 运行镜像即可 秒级启动

Docker 基于Go语言

官网：https://www.docker.com/

文档地址：https://docs.docker.com/

仓库地址：https://hub.docker.com/

Docker作用

虚拟机技术

虚拟机技术缺点：

1.资源占用大

2.冗余步骤多

3.启动慢

容器化技术

不是模拟一个完整的操作系统

两者之间的不同

  • 传统虚拟机，虚拟出硬件，运行完整的操作系统，并在该系统上安装和运行软件
  • 容器内应用则是直接运行在宿主机上，并无虚拟内核以及硬件
  • 容器之间相互隔离，文件独立互不影响

DevOps 开发 运维

应用更快速的交付和部署

传统：帮助文档多，安装程序复杂

Docker：打包镜像、发布测试、一键运行

更便捷的升级和扩容缩容

部署应用简单

镜像即项目，扩展简单

简单系统运维

开发与测试与生成高度一致

更高效的计算资源利用

Docker是内核级别的虚拟化，可在物理机上运行很多容器，服务器性能可以极大的利用

Docker 安装

Docker的基本组成

镜像 image

模版，通过该模板可以创建容器服务，一个镜像可以创建多个容器，最终服务运行在容器当中

容器 container

Docker利用容器技术，独立运行或一组应用，有一系列启停删除基本命令，可以理解为一个简易的系统

仓库 repository

分为公有和私有仓库，存放镜像

Docker Hub

阿里云、腾讯云等，配置镜像加速
